Consignes générales

Vous êtes chargé de développer une application d'analyse de données pour le secteur de la restauration/livraison de plats traditionnels sénégalais. Vous travaillerez sur un dataset MongoDB contenant 5 collections : restaurants, plats, clients, commandes, livreurs.

Environnement technique :
- MongoDB (Docker ou local)
- MongoDB Compass (recommandé pour visualisation)
- Python (Flask/FastAPI) OU PHP (vanilla/Slim)
- HTML/CSS/JavaScript pour l'interface

📌 Objectifs du projet

  1. Importer correctement le dataset MongoDB fourni.
  2. Écrire 10 requêtes d'agrégation MongoDB (niveau expert) répondant aux questions posées.
  3. Développer une mini-application web (Python ou PHP) qui interroge MongoDB et affiche les résultats de manière interactive.
  4. Rédiger un rapport de projet expliquant vos choix techniques et difficultés rencontrées.

10 Questions MongoDB Niveau Expert

Rédigez les requêtes d'agrégation MongoDB correspondant à chaque question. Les requêtes doivent être fonctionnelles et optimisées.

Question 1
Listez tous les restaurants de Dakar avec leurs plats (jointure restaurants → plats). Affichez le nom du restaurant, sa note moyenne, et la liste des plats avec prix. Trier par note décroissante.
Question 2
Pour chaque client, calculez le montant total dépensé, le nombre de commandes, et la moyenne de dépense par commande. Incluez uniquement les commandes au statut "livrée". Classez par total dépensé décroissant.
Question 3
Trouvez les 3 plats les plus commandés (en quantité totale) avec leur prix moyen et leur restaurant d'origine. Utilisez $unwind sur commandes.articles.
Question 4
Calculez le revenu généré par chaque mode de paiement (Mobile Money, Carte, Espèces) et le pourcentage que cela représente du chiffre d'affaires total. Seulement commandes livrées.
Question 5
Identifiez les clients qui ont commandé un plat contenant "tofu" ET un plat contenant "poisson" dans des commandes différentes. (Indices : $lookup, $match, $group, $setIntersection ou $all).
Question 6
Construisez une matrice (dashboard) avec $facet : répartition des commandes par statut + montant total des commandes livrées par mois + top 3 clients fidèles (points_fidelite).
Question 7
Effectuez une recherche géographique avec $geoNear : trouvez les 2 restaurants les plus proches du client Ousmane Sarr (coordonnées -17.480, 14.720). Affichez le nom du restaurant et la distance en mètres.
Question 8
Pour chaque livreur, calculez : son nom, le nombre de commandes livrées, le temps de livraison moyen (delai_livraison_minutes), et la note moyenne du livreur. Seulement commandes avec statut "livrée" et livreur_id non null.
Question 9
Analyse des régimes alimentaires : comptez le nombre de commandes qui contiennent au moins un plat "vegetarien" ou "vegan". Utilisez $lookup entre commandes.articles.plat_id et plats.regime.
Question 10 (Super expert - Bonus)
Créez un système de recommandation de plats pour chaque client : trouvez les plats commandés par des clients similaires (même ville) mais que le client actuel n'a jamais commandés. Utilisez $group + $addToSet + $setDifference. Implémentez pour le client "Ousmane Sarr".

Mini-application Web (Python ou PHP)

Vous devez développer une interface web permettant d'interroger la base MongoDB et d'afficher les résultats des 10 requêtes de manière interactive.

Option Python

Stack recommandée : Flask ou FastAPI + PyMongo ou Motor. Exemple de structure :

# app.py (Flask)
from flask import Flask, render_template, jsonify
from pymongo import MongoClient

app = Flask(__name__)
client = MongoClient('mongodb://localhost:27017/')
db = client.alimentation_senegal

@app.route('/')
def index():
    return render_template('index.html')

@app.route('/api/question/1')
def question1():
    pipeline = [ ... ]  # Votre requête ici
    result = list(db.commandes.aggregate(pipeline))
    return jsonify(result)

if __name__ == '__main__':
    app.run(debug=True)

Option PHP

Stack recommandée : PHP 8+ avec extension MongoDB. Exemple :

<?php
// index.php
require_once __DIR__ . '/vendor/autoload.php';
$client = new MongoDB\Client("mongodb://localhost:27017");
$db = $client->alimentation_senegal;

$question1 = $db->commandes->aggregate([ ... ]); // Votre pipeline

header('Content-Type: application/json');
echo json_encode(iterator_to_array($question1));
?>

Fonctionnalités attendues de l'application web

  • Une page d'accueil avec présentation du projet et du dataset.
  • Un menu ou des onglets pour accéder aux 10 questions.
  • Pour chaque question : affichage de l'énoncé, de la requête MongoDB utilisée, et des résultats formatés (tableau HTML, graphique, ou JSON).
  • Au moins 2 visualisations graphiques (Chart.js, D3.js ou autre) pour les questions 4 et 6.
  • Interface responsive et stylisée (CSS moderne).
  • Code bien structuré, commenté, séparation backend/frontend.
Conseils : Utilisez AJAX/Fetch pour charger les résultats sans recharger la page. Gérez les erreurs (connexion MongoDB, requêtes vides). Pensez à la pagination si certains résultats sont volumineux.
